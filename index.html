Triángulo de Restricciones del Proyecto

📌 Concepto Fundamental

El triángulo de restricciones (también llamado "triple restricción") representa las tres dimensiones interdependientes de todo proyecto: Alcance, Tiempo y Costo. En el centro está la Calidad.

🔺 El Triángulo de Restricciones

⏱️ TIEMPO Duración y Cronograma
📊 ALCANCE Requerimientos y Entregables
💰 COSTO Presupuesto y Recursos
✨ CALIDAD ✨

⚖️ Interdependencia

⬆️ Si aumenta el Alcance

→ Aumenta el Tiempo necesario
→ Aumenta el Costo del proyecto
→ Puede afectar la Calidad

⏩ Si reduces el Tiempo

→ Debes reducir Alcance o aumentar Costo
→ Más recursos = más dinero
→ Riesgo de comprometer Calidad

💵 Si reduces el Costo

→ Debes reducir Alcance o extender Tiempo
→ Menos recursos disponibles
→ Calidad puede verse afectada

✅ Puntos Clave

  • Las tres restricciones están interrelacionadas: cambiar una afecta las otras
  • La Calidad está en el centro y depende del equilibrio de las tres
  • El Project Manager debe gestionar este equilibrio constantemente
  • No se puede optimizar todo simultáneamente - hay trade-offs
  • La gestión exitosa requiere negociación con stakeholders

Earned Value Management (EVM)

📌 ¿Qué es EVM?

Earned Value Management es una metodología de control de proyectos que integra alcance, cronograma y costos para medir el desempeño y progreso del proyecto de manera objetiva.

📊 Conceptos Fundamentales de EVM

Acrónimo Nombre Completo Descripción
PV Planned Value Valor Planificado - Presupuesto autorizado asignado al trabajo programado
EV Earned Value Valor Ganado - Valor del trabajo realmente completado
AC Actual Cost Costo Real - Costo incurrido por el trabajo completado
BAC Budget at Completion Presupuesto al Completar - Presupuesto total del proyecto
CV Cost Variance Variación de Costo - CV = EV - AC
CPI Cost Performance Index Índice de Desempeño de Costo - CPI = EV / AC
SV Schedule Variance Variación de Cronograma - SV = EV - PV
SPI Schedule Performance Index Índice de Desempeño del Cronograma - SPI = EV / PV
EAC Estimate at Completion Estimado al Completar - Pronóstico del costo total final
ETC Estimate to Complete Estimado para Completar - Costo esperado para terminar
VAC Variance at Completion Variación al Completar - VAC = BAC - EAC

📐 Fórmulas Clave de EVM

💚 Variación de Costo (CV)

CV = EV - AC

CV > 0 → Bajo presupuesto ✅
CV < 0 → Sobre presupuesto ❌

🧡 Variación de Cronograma (SV)

SV = EV - PV

SV > 0 → Adelantado ✅
SV < 0 → Atrasado ❌

💙 Índice de Desempeño de Costo (CPI)

CPI = EV / AC

CPI > 1 → Eficiente en costos ✅
CPI < 1 → Ineficiente ❌

💜 Índice de Desempeño de Cronograma (SPI)

SPI = EV / PV

SPI > 1 → Adelantado ✅
SPI < 1 → Atrasado ❌

🔮 Estimado al Completar (EAC)

EAC = BAC / CPI

Pronóstico del costo total final del proyecto basado en desempeño actual

⏭️ Estimado para Completar (ETC)

ETC = EAC - AC

Cuánto dinero más se necesita para terminar el proyecto

✅ Interpretación Rápida de Indicadores

  • CPI y SPI > 1: Proyecto va bien en costo y tiempo ✨
  • CPI < 1: Gastando más de lo planeado - revisar presupuesto
  • SPI < 1: Proyecto atrasado - tomar acciones correctivas
  • CV y SV positivos: Proyecto saludable
  • CV y SV negativos: Proyecto en problemas - intervención urgente
💡 Ejemplo Práctico: Si BAC = $100,000, EV = $40,000, AC = $50,000, PV = $45,000:
• CV = $40,000 - $50,000 = -$10,000 (sobre presupuesto)
• SV = $40,000 - $45,000 = -$5,000 (atrasado)
• CPI = $40,000 / $50,000 = 0.8 (ineficiente)
• SPI = $40,000 / $45,000 = 0.89 (atrasado)
• EAC = $100,000 / 0.8 = $125,000 (proyección final)

EDT - Estructura de Desglose del Trabajo (WBS)

📌 ¿Qué es una EDT/WBS?

EDT (Estructura de Desglose del Trabajo) o WBS (Work Breakdown Structure) es una presentación simple y organizada del trabajo requerido para completar el proyecto, orientada a los entregables o "productos del trabajo" que son el resultado del esfuerzo y no el esfuerzo en sí.

🎯 Concepto Clave: La EDT descompone el proyecto en componentes más pequeños y manejables. Se enfoca en QUÉ se va a entregar, no en CÓMO hacerlo.

🏗️ Características de una EDT

📦 Orientada a Entregables

Se enfoca en los productos, resultados y servicios que se van a crear, no en las actividades para crearlos.

🔽 Jerárquica

Organización en niveles: del más general (proyecto completo) al más específico (paquetes de trabajo).

📊 100% del Proyecto

Incluye TODO el trabajo del proyecto. Si no está en la EDT, no es parte del alcance.

🎁 Paquetes de Trabajo

El nivel más bajo son los paquetes de trabajo, unidades de trabajo asignables y medibles.

🚀 Kick-Off Meeting

🎪 ¿Qué es el Kick-Off?

El Kick-Off Meeting es la reunión de inicio formal del proyecto donde se presenta y aprueba el Plan de Gestión del Proyecto. Marca el comienzo oficial de la ejecución.

👥 Participantes

• Sponsor del proyecto
• Project Manager
• Equipo del proyecto
• Stakeholders clave
• Cliente (si aplica)

📋 Agenda Típica

• Presentación del equipo
• Revisión de objetivos
• Aprobación del plan
• Cronograma y presupuesto
• Roles y responsabilidades
• Plan de comunicación

🎯 Objetivos

• Alinear expectativas
• Establecer compromiso
• Aclarar alcance
• Definir comunicación
• Obtener buy-in del equipo

✅ Puntos Clave EDT y Kick-Off

  • EDT se enfoca en ENTREGABLES (qué), no en actividades (cómo)
  • La EDT incluye el 100% del trabajo del proyecto
  • Descomposición jerárquica: niveles de detalle creciente
  • Paquetes de trabajo son el nivel más bajo y asignable
  • Kick-Off marca el inicio formal de la ejecución
  • En el Kick-Off se aprueba el Plan de Gestión del Proyecto

SCRUM - Metodología Ágil

📌 ¿Qué es SCRUM?

Scrum define un proceso empírico, interactivo e incremental de desarrollo que intenta obtener ventajas mediante la construcción de un marco de trabajo, en el cual las personas pueden acometer problemas complejos adaptativos maximizando el valor de los productos obtenidos de manera creativa a la vez que se mejora el rendimiento del equipo. Se trata de un proceso en el que se aplican un conjunto de buenas prácticas para trabajar de manera colaborativa en equipo.

🎯 Filosofía Ágil: SCRUM se basa en entregas incrementales, adaptación continua, colaboración estrecha y mejora constante.

🏛️ Pilares de SCRUM

👁️ Transparencia

Visibilidad de todos los aspectos del proceso que afectan el resultado. Todos conocen el estado actual.

🔍 Inspección

Revisión frecuente del progreso hacia el objetivo para detectar variaciones no deseadas.

🔄 Adaptación

Ajuste del proceso o producto cuando la inspección determina que hay desviaciones inaceptables.

👥 Roles en SCRUM

🎯 Product Owner (PO)

Responsable del producto:
• Define y prioriza el Product Backlog
• Representa la voz del cliente
• Maximiza el valor del producto
• Acepta o rechaza entregables

🎓 Scrum Master (SM)

Facilitador del proceso:
• Asegura que el equipo siga SCRUM
• Elimina impedimentos
• Protege al equipo de distracciones
• Coach y mentor del equipo

👨‍💻 Development Team

Equipo de desarrollo:
• Auto-organizado y multifuncional
• 3-9 personas idealmente
• Responsables de entregar el incremento
• Sin jerarquías internas

📦 Artefactos de SCRUM

📋 Product Backlog

Lista ordenada y priorizada de todo lo que se necesita en el producto. Es dinámica y evoluciona constantemente.

📝 Sprint Backlog

Subconjunto del Product Backlog seleccionado para el Sprint actual más el plan para entregar el incremento.

✅ Incremento

Suma de todos los elementos del Product Backlog completados durante un Sprint. Debe ser potencialmente entregable.

🔄 Ciclo de SCRUM (Sprint)

1

Sprint Planning

Planear qué se va a hacer en el Sprint

→
2

Daily Scrum

Reunión diaria de sincronización (15 min)

→
3

Sprint Review

Demostrar el incremento a stakeholders

→
4

Sprint Retrospective

Reflexionar y mejorar el proceso

⏱️ Duración del Sprint

Un Sprint típicamente dura de 1 a 4 semanas (más común: 2 semanas). La duración se mantiene consistente durante todo el proyecto.

✅ Puntos Clave de SCRUM

  • Metodología ágil, empírica e incremental
  • Pilares: Transparencia, Inspección, Adaptación
  • Roles: Product Owner, Scrum Master, Development Team
  • Artefactos: Product Backlog, Sprint Backlog, Incremento
  • Eventos: Sprint Planning, Daily, Review, Retrospective
  • Entregas frecuentes de valor al cliente
  • Adaptación continua basada en feedback

Fases del Ciclo de Vida del Proyecto

📌 Ciclo de Vida del Proyecto

El ciclo de vida del proyecto define las fases que conectan el inicio de un proyecto con su fin. Cada fase tiene características, entregables y control específicos.

📊 Las 5 Fases Principales

1

INICIACIÓN

• Project Charter
• Identificar stakeholders
• Definir objetivo
• Autorizar el proyecto

→
2

PLANIFICACIÓN

• Plan de gestión
• EDT/WBS
• Cronograma
• Presupuesto

→
3

EJECUCIÓN

• Realizar el trabajo
• Crear entregables
• Gestionar equipo
• Asegurar calidad

→
4

CONTROL

• Monitorear avance
• Control de cambios
• Acciones correctivas
• Reportes de estado

→
5

CIERRE

• Aceptación formal
• Lecciones aprendidas
• Archivar documentos
• Liberar recursos

📈 Curva de Interacción por Fase

💡 Concepto Importante: Diseño y Planificación pueden empalmar parcialmente, pero generalmente son secuenciales. La planificación detallada requiere un diseño base definido.

⬆️ INICIACIÓN

Interacción: Baja
Recursos: Mínimos
Costo: Bajo
Duración: Corta

📈 PLANIFICACIÓN

Interacción: Media-Alta
Recursos: Moderados
Costo: Medio
Duración: Media

🔥 EJECUCIÓN

Interacción: MUY ALTA
Recursos: Máximos
Costo: Más Alto
Duración: Más Larga

📉 CONTROL

Interacción: Media
Recursos: Moderados
Costo: Medio
Nota: Paralelo a ejecución

⬇️ CIERRE

Interacción: Baja
Recursos: Mínimos
Costo: Bajo
Duración: Corta

🎯 División del Proyecto desde Dirección

📐 Cómo se Divide un Proyecto

Desde la perspectiva de dirección de proyectos, un proyecto se divide en:

🗂️ Por Fases

Cada fase tiene entregables específicos, puntos de control (gates) y criterios de salida.

📦 Por Entregables (EDT)

Desglose jerárquico orientado a los productos del trabajo.

⏱️ Por Tiempo (Cronograma)

Actividades secuenciadas en el tiempo con hitos y dependencias.

👥 Por Responsabilidades

Matriz RACI, equipos funcionales, roles específicos.

✅ Puntos Clave de las Fases

  • 5 fases principales: Iniciación, Planificación, Ejecución, Control, Cierre
  • Cada fase tiene entregables y puntos de control específicos
  • Ejecución y Control corren en paralelo
  • Diseño y Planificación pueden empalmar pero son mayormente secuenciales
  • La interacción es MÁS ALTA durante la ejecución
  • Desde dirección el proyecto se divide por: fases, entregables, tiempo y responsabilidades

Estándares y Metodologías

📐 Estándar

Conjunto de especificaciones técnicas y mejores prácticas profesionales para regular, guiar o definir necesidades demandadas por sociedad y tecnología. Define cómo hay que hacer algo.

🔧 Metodología

Conjunto de procedimientos basados en principios lógicos para alcanzar objetivos. Incluye: medidas, herramientas, roles, técnicas. Define qué hay que hacer ante un problema determinado.

📝 Procedimiento

Serie de pasos documentados necesarios para completar con éxito un proyecto o tarea específica.

🌐 ISO 9000:2015

Sistemas de Gestión de Calidad - Fundamentos y Vocabulario

ISO 9000:2015 establece los fundamentos de los sistemas de gestión de calidad y especifica la terminología aplicable. Define conceptos clave como calidad, gestión, proceso, y establecimiento de objetivos de calidad.

📚 PMBOK - Estándar ANSI

¿Es PMBOK un estándar ANSI? SÍ ✅

La primera edición de PMBOK data de 1996. Poco a poco fue calando en las industrias y en la administración de proyectos, convirtiéndose en un estándar ANSI en el año 2000 con su segunda edición (el único con esta categoría para la Gestión de Proyectos) y reconocido internacionalmente en el IEEE Std 1490-2003.

Actualmente va por la 7ª edición. En esta edición tuvo lugar una fuerte importante evolución del estándar y cuando se empezó a cobrar, ya que la de 1996 había sido gratuita.

✅ Puntos Clave

  • Estándar = especificaciones técnicas (CÓMO hacer)
  • Metodología = procedimientos para objetivos (QUÉ hacer)
  • ISO 9000:2015 define fundamentos de gestión de calidad
  • PMBOK es estándar ANSI desde 2000
  • PMBOK actualmente en 7ª edición

Gestión de Proyectos

📌 ¿Qué es un Proyecto?

Un proyecto es una secuencia bien definida de eventos con un principio y un final, dirigidos a alcanzar un objetivo claro, y realizados por personas dentro de unos parámetros establecidos de tiempo, coste, recursos y calidad.

✅ Características de un Proyecto

• Temporal: Tiene inicio y fin definidos
• Único: Crea un producto/servicio único
• Finito: No es perpetuo
• Específico: Objetivo claro y medible

📊 Parámetros Establecidos

• Tiempo: Cronograma definido
• Costo: Presupuesto limitado
• Recursos: Personas y materiales
• Calidad: Estándares a cumplir

📋 Project Charter (Acta de Constitución)

¿Qué es el Project Charter?

Documento que autoriza formalmente el proyecto y otorga al Project Manager la autoridad para asignar recursos. Es la piedra angular que establece la base del proyecto.

📄 Contenido del Project Charter

• Propósito y justificación
• Objetivos medibles
• Requisitos de alto nivel
• Riesgos de alto nivel
• Resumen del presupuesto
• Stakeholders clave
• Autorización del PM

🎯 Importancia

• Autoriza el proyecto formalmente
• Define el PM y su autoridad
• Establece el compromiso organizacional
• Documenta necesidades del negocio
• Sirve como referencia durante el proyecto

🔍 Estudio de Factibilidad

💡 Concepto: Antes de crear el Project Charter, es necesario realizar un estudio de factibilidad que evalúe si el proyecto es viable técnica, económica y operacionalmente.

💰 Factibilidad Económica

¿Es financieramente viable? ROI, beneficio vs costo, presupuesto disponible

🔧 Factibilidad Técnica

¿Tenemos la tecnología y conocimiento? Recursos técnicos, capacidades del equipo

⚙️ Factibilidad Operacional

¿Funcionará en la operación? Impacto organizacional, aceptación de usuarios

✅ Puntos Clave

  • Proyecto = secuencia definida con inicio y fin, objetivo claro
  • Características: temporal, único, finito, específico
  • Project Charter autoriza formalmente el proyecto
  • Estudio de factibilidad previo al Project Charter
  • Factibilidad: económica, técnica y operacional

Gestión de Riesgos

📌 ¿Qué es un Riesgo?

El riesgo es el potencial que una acción o una actividad elegida implique una pérdida o un resultado no deseado. Incluso, la pérdida potencial puede ser considerada como un riesgo.

🎯 Concepto Clave: Riesgo = Incertidumbre que puede afectar los objetivos del proyecto (positiva o negativamente)

🛡️ Gestión de Riesgos

¿Qué implica gestionar riesgos?

Gestionar el riesgo NO significa evitar todos los riesgos. La gestión del riesgo no puede eliminar los riesgos, y tampoco puede prevenir todos los fracasos y malos resultados. Pero puede minimizar las posibilidades de que ocurran, así como reducir el impacto que puedan tener. Puede también ayudar a que el proyecto se recupere de una manera más ágil de los problemas ocurridos inesperadamente.

🔍 Identificación

• Brainstorming con equipo
• Análisis de documentación
• Técnica Delphi
• Checklist de riesgos
• Lecciones aprendidas

📊 Análisis

Cualitativo:
• Probabilidad (Alta/Media/Baja)
• Impacto (Alto/Medio/Bajo)
• Matriz de riesgos

Cuantitativo:
• Análisis numérico
• Simulación Monte Carlo

🎯 Respuesta

Amenazas:
• Evitar • Transferir
• Mitigar • Aceptar

Oportunidades:
• Explotar • Mejorar
• Compartir • Aceptar

📈 Monitoreo

• Seguimiento continuo
• Identificar nuevos riesgos
• Revisar efectividad de respuestas
• Actualizar registro de riesgos
• Reportar status

🔑 Clave para Tratar Riesgos

La clave para tratar con un riesgo es:
1️⃣ Entenderlo - Identificar y analizar
2️⃣ Anticiparlo - Planear antes de que ocurra
3️⃣ Tener un plan de cómo tratarlo - Estrategia de respuesta definida

✅ Puntos Clave

  • Riesgo = potencial de pérdida o resultado no deseado
  • Gestionar riesgos NO es evitarlos todos
  • Objetivo: minimizar probabilidad e impacto
  • Proceso: Identificar → Analizar → Planear Respuesta → Monitorear
  • Clave: entender, anticipar y tener plan de respuesta
  • Respuestas a amenazas: Evitar, Transferir, Mitigar, Aceptar
